A Little About Me

I am a web developer currently living in Columbus, Ohio and love learning about technology and developing web applications. I have worked with Java, Spring and more recently JavaScript.

I've explored lots of different technologies. I am definitely always on the lookout for new shiny things to learn. However I do have sort of a underlying theme to my career. Its usually been about web development and learning things at a deep level so I can get the most out of the framework, technique or language.

It's All About the Web

  • JavaScript

    Early in my career doing Java and JSP, I learned enough JavaScript to get what I needed done. Once I started developing Angular applications, I realized I needed to learn JavaScript as a programming language.

    I have dabbled in Haskell and Scheme, but not using them on a day to day basis I never felt I got very far. I then saw people doing functional programming in JavaScript like Professor Frisby's Mostly Adequate Guide to Functional Programming and I was hooked.

  • REST

    I love REST and API development. My views probably are can be summed up best by the article Create More Web by Panda Strike.

  • Web Components

    • On the front end I am a strong proponent of modular code, especially components. I think any technology or framework that gets closer to enabling Web Components is on the right track. This is a view I have had for awhile. I first came across it with the book Modular Web Design by Nathan Curtis.

  • Semantic Web

    I became interested in the semantic web and resource oriented architecture after attending a NFJS conference and listening to Brian Sletten speak about Resource Oriented Architecture. I had worked with Apache Jack Rabbit on a previous project, but didn't quite 'get' it. This brought it all together. I saw it as a superset of REST, with Resources as the common thread.
